# 场景：物理控制系统

适用于机器、设备、飞行器、机器人、工业过程以及其他存在真实传感器和执行器的系统。这是工程控制论最直接的应用领域。

## 核心对象

| 元素 | 物理系统中的含义 |
|---|---|
| 被控对象 | 电机、飞行器、锅炉、机械臂等 |
| 目标 | 位置、速度、温度、压力、轨迹等 |
| 传感器 | 测量真实状态，但存在噪声和误差 |
| 控制器 | 根据目标与测量值的偏差计算动作 |
| 执行器 | 电机、阀门、舵面、加热器等 |
| 扰动 | 风、负载、摩擦、环境和部件老化 |

## 设计重点

1. 明确允许误差、响应时间和安全边界；
2. 校准传感器与执行器，保留现实误差的调整参数；
3. 分析采样延迟、执行延迟和控制强度是否引发振荡；
4. 在传感器失真、执行器卡死和断电时进入安全状态；
5. 先通过模型、仿真和受控实验验证，再扩大实际运行范围。

## 常见失败

- 只按理论参数设计，没有现场校准；
- 传感器噪声导致控制器频繁反向动作；
- 控制动作过强，系统在目标两侧持续振荡；
- 执行器已经饱和，控制器仍不断累积动作；
- 失去测量后继续运行，而不是停止或降级。

## 最小检查

- 目标和允许误差是什么？
- 传感器精度、频率和延迟是多少？
- 执行器的范围、速度和失效方式是什么？
- 最大扰动下系统是否仍可恢复？
- 失去控制时，哪个物理状态最安全？

物理控制涉及人身或设备安全时，这份文档只能作为思考入口，不能替代相关专业计算、实验、标准和有资质人员的审查。
