# 场景：物理控制系统

适用于机器、设备、飞行器、机器人、工业过程以及其他存在真实传感器和执行器的系统。这是工程控制论最直接的应用领域。

涉及人身、设备或环境风险时，Agent 默认只能分析、仿真和生成审查清单。没有有资质人员批准、现场隔离措施和本次明确执行授权，不得连接、启动或调整真实执行器。

## 核心对象

| 元素 | 物理系统中的含义 |
|---|---|
| 被控对象 | 电机、飞行器、锅炉、机械臂等 |
| 目标 | 位置、速度、温度、压力、轨迹等 |
| 传感器 | 测量真实状态，但存在噪声和误差 |
| 控制器 | 根据目标与测量值的偏差计算动作 |
| 执行器 | 电机、阀门、舵面、加热器等 |
| 扰动 | 风、负载、摩擦、环境和部件老化 |

## 设计重点

1. 明确允许误差、响应时间和安全边界；
2. 校准传感器与执行器，保留现实误差的调整参数；
3. 分析采样延迟、执行延迟和控制强度是否引发振荡；
4. 在传感器失真、执行器卡死和断电时进入安全状态；
5. 先通过模型、仿真和受控实验验证，再扩大实际运行范围。

## 进入真实执行的硬门槛

在接通真实执行器前，必须形成一份可检查的运行合同，并逐项满足：

- 写明允许工况、禁止区域、采样与总延迟上限、执行器幅值与变化率上限；
- 用仿真或受控实验覆盖正常工况、边界工况和已知故障，结果可追溯；
- 急停、限位或联锁已经在真实硬件上验证，且不依赖主控制器正常运行；
- 明确现场负责人、启动授权、监护方式和停止条件；
- 写明控制与监测周期、校准有效期，以及按总纲四种终态裁决的放行和停止证据；
- 任一项证据缺失或当前状态不可确认时，不进入自动运行。

安全层必须独立于性能控制回路。主控制器负责靠近目标；独立安全层负责越界时限幅、断能或急停，不能只靠同一软件进程自我保护。

按故障模式指定安全路径，而不是笼统写“进入安全状态”：

| 故障 | 必须经危害分析确认的候选路径 |
|---|---|
| 传感器失真、失联或观测冲突 | 冻结危险动作，切换可信冗余；否则受控停机 |
| 执行器卡死、饱和或响应异常 | 切断控制输出并隔离能量，转人工处置 |
| 通信、计算或供电中断 | 由独立联锁进入断能、制动或设备定义的安全位 |
| 状态超出已验证工况 | 立即退出自动控制，不用未验证模型继续追踪目标 |

具体安全路径必须由设备危害分析确定；如果断能本身会制造更大危险，就采用经验证的受控降级或安全停机序列。

## 常见失败

- 只按理论参数设计，没有现场校准；
- 传感器噪声导致控制器频繁反向动作；
- 控制动作过强，系统在目标两侧持续振荡；
- 执行器已经饱和，控制器仍不断累积动作；
- 失去测量后继续运行，而不是停止或降级。

## 最小检查

- 目标和允许误差是什么？
- 传感器精度、频率和延迟是多少？
- 执行器的范围、速度和失效方式是什么？
- 最大扰动下系统是否仍可恢复？
- 失去控制时，哪个物理状态最安全？
- 运行合同是否完整，独立安全层是否在真实硬件上验证？

物理控制涉及人身或设备安全时，这份文档只能作为思考入口，不能替代相关专业计算、实验、标准和有资质人员的审查。
