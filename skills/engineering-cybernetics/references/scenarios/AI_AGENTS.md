# 场景：AI Agent 与自动化

适用于能够读取环境、作出判断并调用工具改变外部状态的 AI Agent。Agent 的输出存在不确定性，因此控制重点不是让它“永远正确”，而是限制错误、验证结果并保留人的控制权。

## 典型闭环

```text
目标：完成一个明确且可验证的任务
观测：输入、工具返回值、系统状态和验证结果
动作：搜索、生成、修改、调用工具或停止
验证：独立检查真实外部状态，而不是相信模型自述
安全状态：停止执行、回滚可逆变化、请求人工决定
```

## 设计重点

1. 把目标写成可验收结果，不用宽泛愿景授权连续行动；
2. 将模型建议与外部事实分开，工具结果优先于模型信心；
3. 高风险动作设置权限、预算、范围和次数边界；
4. 动作完成后检查真实状态，不把“工具调用成功”当成业务成功；
5. 重复失败时停止并暴露阻塞，不无限重试或悄悄缩小目标；
6. 多 Agent 协作时明确目标所有者、共享状态和写入边界；
7. 涉及方向、不可逆代价或用户未授权范围时，由人决定。

## 常见失败

- Agent 报告“已完成”，但没有验证外部结果；
- 为完成模糊目标不断扩张任务范围；
- 多个 Agent 根据旧状态同时修改同一对象；
- 工具返回异常后重复调用，扩大费用或副作用；
- 把多个模型意见一致当成事实证明；
- 为提高自主性移除停止条件和人工确认。

## 最小检查

- Agent 可以改变哪些外部状态？
- 哪些结果必须通过工具或测试独立验证？
- 单次与累计的时间、费用和动作上限是什么？
- 什么失败需要停止，什么情况需要人工介入？
- 多 Agent 如何共享最新状态并避免冲突？

Agent 的自主性应来自可靠闭环，而不是来自更宽泛的权限。
