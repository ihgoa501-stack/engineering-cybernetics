# Engineering Cybernetics — Agent 运行规则

在非简单的软件建设、自动化、AI Agent、业务闭环或物理控制任务中，使用工程控制论检查目标、状态、反馈与停止条件。一次性、低风险、没有持续状态变化的任务，不要强行套用控制论。

## 开始工作

1. 阅读 `ENGINEERING_CYBERNETICS.md`；
2. 写清控制目标、约束、当前状态、观测信号、可用动作、主要扰动、结果反馈和安全状态；
3. 阅读 `SCENARIOS.md` 并选择一个主场景；存在独立的跨域风险时，再补读其他场景；
4. 先完成一个可观察、可纠偏的小闭环，不因框架而扩大产品范围；
5. 修改后验证真实结果，并明确区分已验证事实、推断和未知。

## 委派与子 Agent

委派时必须把当前控制目标、约束、可验证结果和停止条件写入子任务，并要求子 Agent 阅读本文件及适用场景。子 Agent 的一致意见只是参考信号，不能提高事实的证据等级。

## 边界

- 控制论帮助系统靠近目标，不替用户决定目标是否值得追求。
- 文档是决策框架，不是事实证明，也不是安全标准的替代品。
- 规则指导行为；必须执行的权限、测试和危险操作限制仍应由 Hook、权限或 CI 强制。
